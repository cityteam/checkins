Checkin Each Guest

Now, we are ready to assign a mat for our first Guest. Click on any of the rows that do not currently have a Guest assigned (we will use mat 2 here), and see what happens next.

The first person comes up and says his name is Barney Rubble (no real names in this example!). It is important to see if this Guest has stayed at this Facility before. Let's search all previous Guests to see if we can find a match. You can search on any set of letters within the name (upper case or lower case does not matter). So, let's search for ub and see what we get.

As you type each character, the list of matches will change. If there are two many matches for one page, the pagination controls (< or >) will light up, allowing you to page through the various matches.

You can also search for part of the first name OR part of the last name, by typing a space between the arts. For example, ar ub would also find this person.

We see that Barney has checked in here before, so click his name to complete the assignment.

The first two fields, Payment Type and Payment Amount, are required. The Payment Type field indicates whether the person is paying in cash ($$), or for whatever other reason (see the dropdown for options) they do not have to pay tonight. The payment amount defaults to $5 because that is what CityTeam Portland currently requests -- erase it if this person is not paying.

You can also optionally enter a time this person wants to take a shower, or just gets woken up in the morning. Use HH:MM or HH:MM:SS format for these. You can also add a comment like Will arrive from work after 10:00 if needed.

When you are through with this assignment, click to proceed.

Back at the Manage Checkins page, we see that this Guest has indeed been assigned to Mat 2, and the totals at the bottom have been updated. This will occur every time a change is made. The Total $$ amount is also useful in checking how much cash was actually received, versus what you expected.

If you need a hard copy of the checkin information at the end of the process, or at any other time, simply print the current page (File -> Print).

Now, the next person (Fred Flintstone) is ready to check in, and we wish to assign him to mat number 4.

Click anywhere on the row for mat 4, and search using fred (remember, it does not matter whether you use upper case or lower case). But this time, we do not find him - this is the first time he has checked in at this Facility.

To add a new Guest, simply click the button.

Fill out the details for the new Guest, and click the button. The application will save the new Guest information, start the assignment to the currently selected mat (4), and proceed to Step 2 as we did in the case of the first Guest.

NOTE: The application will check whether this first name / last name combination really is unique (for this particular Facility). If not, you will need to make an adjustment before you can save and proceed.

Follow the Adjust Existing Checkin below to find out how to adjust existing assignments when needed.
